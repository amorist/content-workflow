## 一、爆款公式

```
节点/禁忌/机会 + 人群/空间 + 动作 + 结果 + 产品载体
```

### 公式拆解

| 元素 | 说明 | 作用 |
|------|------|------|
| **节点/禁忌/机会** | 时间节点、禁忌提醒、机会窗口 | 制造紧迫感和行动理由 |
| **人群/空间** | 精准人群标签、使用场景 | 让目标用户产生代入感 |
| **动作** | 简单可执行的具体动作 | 降低行动门槛 |
| **结果** | 行动后的状态变化 | 描绘美好未来 |
| **产品载体** | 承载解决方案的产品 | 自然植入带货 |

### 公式示例

> **完整句式：**
> "立秋前（节点），所有女生（人群）一定要把包整理一下（动作），状态会越来越好（结果），这个香包可以帮你（产品）"

## 二、五大底层逻辑

### 1. 卖时机 ⏰

**核心：** 用时间节点制造行动理由

**常用节点：**
- 节气：立秋、三伏天、冬至、立春
- 时段：年初、月底、换季、节前
- 时机：最近、这段时间、如果你最近

**话术模板：**
- "XX前一定要做这个动作"
- "XX千万别忽视这个细节"
- "最近这段时间，XX要注意"

---

### 2. 卖身份 👤

**核心：** 精准人群代入，让用户觉得"说的就是我"

**人群标签：**
- 性别：所有女生、男生、宝妈
- 职业：做生意的人、上班族、创业者
- 状态：状态不太好的人、经常出门的人
- 家庭：家里有孩子的、已婚女性

**话术模板：**
- "所有女生一定要注意这个细节"
- "做生意的女人要记住这件事"
- "如果你也是XX的人"

---

### 3. 卖解释 🧠

**核心：** 重新定义问题，给出新的认知框架

**解释类型：**
- 概念替换：包=米库、旧物=拖累
- 原因重构：不是你不努力，是方法不对
- 认知升级：你以为A，其实是B

**话术模板：**
- "你以为赚得少，其实是留不住"
- "你以为家里乱，其实是旧物问题"
- "包不只是包，它是你的米库"

---

### 4. 卖动作 ✋

**核心：** 简单可执行的动作，降低行动门槛

**动作类型：**
- 放置类：放、挂、摆、贴
- 使用类：喷、戴、喷、整理
- 清理类：清、换、扔、整理

**话术模板：**
- "把XX放在包里"
- "出门前喷一下"
- "挂一个在衣柜"

---

### 5. 卖结果 ✨

**核心：** 状态与未来的想象，描绘美好画面

**结果类型：**
- 状态改善：整个人很轻松、状态很干净
- 运势提升：越来越顺、状态越来越好
- 问题解决：不再烦躁、眼睛不干涩

**话术模板：**
- "状态会越来越好"
- "整个人更顺"
- "用过都来报喜"

## 三、完整链路

```
节点 → 人群 → 问题 → 解释 → 动作 → 产品 → 案例 → 收口
```

### 链路说明

| 步骤 | 目的 | 示例 |
|------|------|------|
| **节点** | 抓住时机 | "立秋前" |
| **人群** | 精准定位 | "所有女生" |
| **问题** | 戳中痛点 | "钱留不住" |
| **解释** | 重构认知 | "包是米库" |
| **动作** | 给出方案 | "把包整理一下" |
| **产品** | 自然植入 | "这个香包" |
| **案例** | 建立信任 | "很多粉丝用了反馈很好" |
| **收口** | 促进行动 | "赶紧用起来" |

## 四、快速套用模板

### 模板1：节点+人群+痛点+解释+动作

> **结构：** 在[节点]，[人群]一定要[动作]，因为[解释]，解决[痛点]

**示例：**
- "立秋前，所有女生一定要把包整理一下，因为包是米库，解决钱留不住的问题"
- "三伏天，做生意的人一定要挂一个香片，因为味道影响状态，解决状态很散的问题"

### 模板2：反常识+解释+动作+结果

> **结构：** 你以为[表象]，其实是[本质]，[动作]，[结果]

**示例：**
- "你以为赚得少，其实是留不住，把包整理一下，状态会越来越好"
- "你以为家里乱，其实是旧物问题，把旧物清掉，整个人很轻松"

### 模板3：疑问+解释+动作+产品

> **结构：** 为什么[现象]？因为[解释]，[动作]，[产品]

**示例：**
- "为什么你总觉得不顺？因为环境影响人，把空间收纳好，这个香薰帮你"
- "为什么家里总是很烦躁？因为旧物拖状态，把旧物清掉，用这个收纳"

## 五、使用指南

### 生成步骤

1. **选节点** - 确定时间切入点
2. **定人群** - 明确目标用户
3. **找痛点** - 挖掘核心问题
4. **给解释** - 提供新认知
5. **设动作** - 设计简单动作
6. **配产品** - 自然植入产品
7. **加案例** - 增强可信度
8. **做收口** - 促进行动

### 检查清单

- [ ] 是否有明确的节点/时机？
- [ ] 人群是否精准具体？
- [ ] 痛点是否戳中用户？
- [ ] 解释是否有说服力？
- [ ] 动作是否简单可执行？
- [ ] 产品植入是否自然？
- [ ] 是否有案例支撑？
- [ ] 收口是否有紧迫感？
