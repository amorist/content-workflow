## 工作流命令

### 1. 记录选题
**触发词：** 「记录选题：XXX」或「记个选题」

**执行步骤：**
1. 解析选题内容（主题、角度、核心观点）
2. 使用 `lark-cli` 写入飞书多维表格：
   ```bash
   lark-cli api POST /open-apis/bitable/v1/apps/MAr1bOFujaC53FssmwNcgnGqnMe/tables/tblEXW02QlfdHT4S/records \
     --data '{"fields":{"标题":"...","状态":"🆕 新选题","标签":["AI","干货"],"摘要":"...","核心观点":"...","背景/灵感来源":"...","预计受众":"...","可延伸方向":"..."}}'
   ```
3. 同步在「选题进度追踪」表添加一条记录
4. 回复确认

**参考：** [references/选题记录.md](references/选题记录.md)

---

### 2. 检索素材
**触发词：** 「检索素材：XXX」或「搜一下素材」或「找找有没有」

**执行步骤：**
1. 分析检索主题，提取关键词
2. 按顺序检索：
   - `01-素材库/核心概念库/` — 相关理论框架
   - `01-素材库/金句库/` — 相关精彩表达
   - `01-素材库/案例库/` — 相关案例故事
   - `01-素材库/结构库/` — 相关内容结构
   - `01-素材库/爆款拆解/` — 类似爆款分析
   - `03-文稿库/定稿/` — 以往类似主题
3. 汇总检索结果，给出「素材拼接建议」
4. 如果找到高匹配内容，明确问用户：「是否复用？」

**参考：** [references/检索素材.md](references/检索素材.md)

---

### 3. 生成文稿
**触发词：** 「生成文稿：XXX」或「写个脚本」

**执行步骤：**
1. 如果用户未提供选题，先问清楚主题
2. 先执行「检索素材」步骤
3. 基于素材生成脚本
4. 输出格式：
   ```
   # 《{标题}》
   
   ## 结构框架
   [1分钟口播结构]
   
   ## 完整脚本
   【开场钩子】
   {前5秒强开屏}
   
   【正文】
   {主体内容 - 分为3-4个要点}
   
   【结尾引导】
   {行动号召/互动引导}
   
   ## 时长估算
   总计：约X分钟
   ```
5. 写入 `03-文稿库/草稿/{标题}_{日期}.md`

**参考：** [references/生成文稿.md](references/生成文稿.md)

---

### 4. 优化开头
**触发词：** 「优化开头」或「前5秒」或「开屏」

**执行步骤：**
1. 读取当前文稿开头
2. 基于「短视频开头方法论」给出3个方案：
   - 方案A：「痛点开屏」— 戳用户痛点
   - 方案B：「好奇开屏」— 制造悬念/反差
   - 方案C：「数字开屏」— 用数据抓注意力
3. 每个方案说明核心逻辑和适用场景
4. 用户选完后，更新文稿

**参考：** [references/优化开头.md](references/优化开头.md)

---

### 5. 生成标题
**触发词：** 「生成标题」或「起个标题」或「标题」

**执行步骤：**
1. 读取文稿核心内容
2. 基于「短视频标题方法论」生成5个标题
3. 每个标题注明类型：
   - 痛点型：「{问题}，{解决方案}」
   - 好奇型：「{反差/悬念}」
   - 数字型：「{数字}个{主题}`
   - 利益型：「{收益}的{方法}`
4. 用户选定后更新文稿

**参考：** [references/生成标题.md](references/生成标题.md)

---

### 6. 数据复盘
**触发词：** 「记录数据」或「数据复盘」或「发布后」

**执行步骤：**
1. 用户输入：标题 / 播放量 / 点赞 / 评论 / 收藏
2. 写入 `04-发布与数据/内容数据记录.md`
3. 自动判断是否值得加入爆款拆解（播放>5000 或 点赞>500）
4. 如果是爆款：
   - 生成爆款拆解到 `01-素材库/爆款拆解/`
   - 更新 `05-方法论沉淀/选题规律.md`
5. 生成简短复盘

**参考：** [references/数据复盘.md](references/数据复盘.md)

## 内容创作核心原则

1. **复用 > 从零创作**：每次先检索，用户已有积累是最大资产
2. **结构化思维**：结论先行，要点清晰，口语化表达
3. **数据驱动**：记录每次发布数据，让方法论「活」起来
4. **素材积累**：好内容及时沉淀到素材库

## 初始启动

如果素材库为空，主动提示用户：
- 「你的素材库现在还是空的。建议先做：
  1. 导入3-5个你觉得好的内容/案例
  2. 把你的第一条视频脚本作为初始文稿
  3. 建立2-3个核心概念卡片
  这样AI才能帮你真正复用和检索。」
