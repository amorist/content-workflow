# 短视频开头优化方法论

## 前5秒为什么重要

- 平台算法看的是「完播率」和「互动率」
- 前5秒决定用户是否继续看
- 黄金开屏 = 注意力抓取 + 价值承诺

## 四种强开屏模式

### 1. 痛点开屏
**公式：** {描述一个让人难受的场景/问题}
**目的：** 「这说的就是我！」→ 继续看找解决方案
**示例：**
- 「你是不是也这样？写一篇文章憋了3小时」
- 「每天加班到12点，工作还是做不完」
- 「AI那么火，但用起来发现根本没那么多场景」

### 2. 好奇开屏
**公式：** {说一个反常识的事实/制造悬念}
**目的：** 「真的吗？不可能吧？」→ 想知道答案
**示例：**
- 「我用AI写稿，阅读量反而更高了」
- 「不是ChatGPT不行，是你问的方式不对」
- 「99%的人都用错了AI，包括你」

### 3. 数字开屏
**公式：** {具体数字 + 惊人效果}
**目的：** 「什么方法这么厉害？」→ 想学方法
**示例：**
- 「我把写稿时间从3小时缩到20分钟，就靠这3步」
- 「用好这个AI工具，效率提升10倍」
- 「3个月，我用AI赚了8万」

### 4. 成果开屏
**公式：** {展示一个让人眼红的成果}
**目的：** 「怎么做到的？」→ 想学方法
**示例：**
- 「看这是我用AI做的第100个项目」
- 「这是我今天用AI赚到的第1000块」
- 「我的AI工作流，让我同时做3份工作」

## 开头优化流程

1. 读取当前文稿开头
2. 判断当前开头属于哪种类型
3. 针对同一主题，生成其他3种开屏方案
4. 解释每种方案的心理逻辑
5. 让用户选择，确定后更新文稿

## 优化输出格式

```markdown
## 当前开头分析

**当前开头：** 「{原文}」
**类型：** {痛点/好奇/数字/成果}
**问题：** {指出现有开头的问题}

---

## 优化方案

### 方案A：痛点开屏
**文案：**
「{痛点开屏文案}」

**心理逻辑：** 通过描述目标用户的痛苦场景，让他们产生「说的就是我」的共鸣，从而继续看下去找解决方案。

**适用场景：** 受众痛点明确、内容提供解决方案

---

### 方案B：好奇开屏
**文案：**
「{好奇开屏文案}」

**心理逻辑：** 通过反常识或制造悬念，激发用户的好奇心，让他们想知道「为什么」或「怎么做」。

**适用场景：** 有独特见解、内容有反差感

---

### 方案C：数字开屏
**文案：**
「{数字开屏文案}」

**心理逻辑：** 通过具体数字制造「可信感」和「利益感」，让用户觉得这条内容值得看。

**适用场景：** 有具体数据/案例、内容偏实用方法论
```

## 选择建议

| 类型 | 适合场景 | 注意事项 |
|------|---------|---------|
| 痛点开屏 | 干货教程类、解决方案类 | 痛点要精准，别太大太空 |
| 好奇开屏 | 观点输出类、有独特见解 | 不要过度夸张，会降低信任 |
| 数字开屏 | 方法论类、有具体成果 | 数字要真实，不要标题党 |
| 成果开屏 | 成果展示类、案例分享类 | 成果要有说服力，不要假大空 |
