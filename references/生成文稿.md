# 生成文稿工作流

## 触发条件

- 「生成文稿」
- 「写个脚本」
- 「创作内容」
- 「写文案」
- 用户确认选题后

## 执行步骤

### Step 1: 运行前置检查

```bash
bash content-workflow/.content-workflow/preamble.sh
```

### Step 2: 确认内容类型（必须）

如果用户未指定内容类型，**必须询问**：

> 请确认要生成哪种类型的内容：
> 
> 1. **口播视频** — 利他内容、知识分享、建立信任
> 2. **自然流带货** — 软植入、场景化推荐、用户体验
> 3. **投流视频** — 强转化、直接卖货、促销引导

根据用户选择，加载对应的策略和模板。

### Step 3: 6个自动决策

根据选题和用户配置，自动决策：

| 决策项 | 决策逻辑 | 输出 |
|-------|---------|------|
| **1. 选题类型识别** | 分析选题关键词 | 情感/知识/娱乐/带货 |
| **2. 受众画像匹配** | 基于选题推断 | 年龄/性别/痛点 |
| **3. 平台特性适配** | 读取用户配置 | 时长/节奏/风格 |
| **4. 结构模板选择** | 基于内容类型 | 开头/中间/结尾模板 |
| **5. 历史数据参考** | 检索 learnings | 复用高互动元素 |
| **6. 发布时间建议** | 基于平台规律 | 最佳发布时段 |

### Step 4: 全面素材检索

按顺序检索所有相关素材：

```
1. 01-素材库/核心概念库/ — 相关理论框架
2. 01-素材库/金句库/ — 相关精彩表达
3. 01-素材库/案例库/ — 相关案例故事（按主题匹配）
4. 01-素材库/产品资料库/ — 产品信息（如用户提及产品）
5. 01-素材库/结构库/ — 相关内容结构
6. 01-素材库/爆款拆解/ — 类似爆款分析
7. 05-方法论沉淀/ — 方法论指导
8. .content-workflow/learnings.jsonl — 历史经验
```

### Step 4.5: 产品信息智能处理（如适用）

**检测产品信号：**
- 用户输入中包含潜在产品名称
- 提及价格（数字 + 元/块/￥）
- 描述产品卖点/功效
- 明确说"给XX产品写文案"

**处理流程：**

```
检测到产品信息
  ↓
提取产品名称
  ↓
检索产品资料库
  ├─ 产品存在 → 读取完整资料
  │             ↓
  │             标记产品信息来源
  │             更新使用计数
  │
  └─ 产品不存在 → 背景采集
                  ↓
                  提取输入中的产品信息
                  创建基础产品档案
                  标记为"待完善"
                  继续生成（不中断）
```

**产品信息融合：**

如用户提供了产品名称，在生成文案时：

**投流视频：**
- 开头 → 结合产品解决的核心痛点
- 中段 → 融入产品卖点 + 用户收益 + 差异化优势
- 结尾 → 价格信息 + 促销话术 + CTA

**自然流带货：**
- 场景切入 → 使用产品的真实场景
- 体验描述 → 结合产品功能特点 + 用户证言
- 软性推荐 → 自然引出产品

**口播视频：**
- 知识分享 → 结合产品相关的知识点
- 案例植入 → 产品作为案例或解决方案
- 价值输出 → 产品带来的改变和收益

**占位符自动替换：**
- `{产品名称}` → 实际产品名称
- `{价格}` → 产品当前价格
- `{原价}` → 产品原价（如有）
- `{核心卖点}` → 产品核心卖点列表
- `{目标人群}` → 产品目标人群
- `{使用场景}` → 产品使用场景
- `{用户证言}` → 产品用户证言

### Step 8: 保存文稿

保存到草稿库：

```bash

EOF
```

## Boil the Lake 原则

当生成文稿时，**不要只做最小可行**，而是：

- ✅ 给完整脚本，不是大纲
- ✅ 给5个标题选项，不是1个
- ✅ 给画面建议，不是纯文字
- ✅ 给违规检查报告，不是假设没问题
- ✅ 给复用素材清单，方便追溯

## 质量检查清单

生成完成后，自检：

- [ ] 开头前5秒是否有强钩子？
- [ ] 正文是否有具体案例/数据？
- [ ] 结尾是否有明确的CTA？
- [ ] 语言是否口语化、易听懂？
- [ ] 是否符合平台调性？
- [ ] 是否通过违规检查？
- [ ] 是否有明确的受众指向？
