# 生成文稿工作流

## 触发条件

- 「生成文稿」
- 「写个脚本」
- 「创作内容」
- 「写文案」
- 用户确认选题后

## 执行步骤

### Step 1: 运行前置检查

```bash
bash /Users/amor/Documents/智伴同行/AI知识付费/content-workflow/.content-workflow/preamble.sh
```

### Step 2: 确认内容类型（必须）

如果用户未指定内容类型，**必须询问**：

> 请确认要生成哪种类型的内容：
> 
> 1. **口播视频** — 利他内容、知识分享、建立信任
> 2. **自然流带货** — 软植入、场景化推荐、用户体验
> 3. **投流视频** — 强转化、直接卖货、促销引导

根据用户选择，加载对应的策略和模板。

### Step 3: 6个自动决策

根据选题和用户配置，自动决策：

| 决策项 | 决策逻辑 | 输出 |
|-------|---------|------|
| **1. 选题类型识别** | 分析选题关键词 | 情感/知识/娱乐/带货 |
| **2. 受众画像匹配** | 基于选题推断 | 年龄/性别/痛点 |
| **3. 平台特性适配** | 读取用户配置 | 时长/节奏/风格 |
| **4. 结构模板选择** | 基于内容类型 | 开头/中间/结尾模板 |
| **5. 历史数据参考** | 检索 learnings | 复用高互动元素 |
| **6. 发布时间建议** | 基于平台规律 | 最佳发布时段 |

### Step 4: 全面素材检索

按顺序检索所有相关素材：

```
1. 01-素材库/核心概念库/ — 相关理论框架
2. 01-素材库/金句库/ — 相关精彩表达
3. 01-素材库/案例库/ — 相关案例故事（按主题匹配）
4. 01-素材库/结构库/ — 相关内容结构
5. 01-素材库/爆款拆解/ — 类似爆款分析
6. 05-方法论沉淀/ — 方法论指导
7. .content-workflow/learnings.jsonl — 历史经验
```

### Step 5: 加载方法论

**口播视频：**
- 05-方法论沉淀/短视频开头方法论.md

- 05-方法论沉淀/爆款文案核心公式.md

**自然流带货：**
- 05-方法论沉淀/短视频开头方法论.md
- 05-方法论沉淀/短视频三段式方法论.md
- 05-方法论沉淀/文案素材库.md

**投流视频：**
- 05-方法论沉淀/爆款文案核心公式.md
- 05-方法论沉淀/短视频三段式方法论.md

## Boil the Lake 原则

当生成文稿时，**不要只做最小可行**，而是：

- ✅ 给完整脚本，不是大纲
- ✅ 给5个标题选项，不是1个
- ✅ 给画面建议，不是纯文字
- ✅ 给发布时间建议，不是只给内容
- ✅ 给违规检查报告，不是假设没问题
- ✅ 给复用素材清单，方便追溯

## 质量检查清单

生成完成后，自检：

- [ ] 开头前5秒是否有强钩子？
- [ ] 正文是否有具体案例/数据？
- [ ] 结尾是否有明确的CTA？
- [ ] 语言是否口语化、易听懂？
- [ ] 是否符合平台调性？
- [ ] 是否通过违规检查？
- [ ] 是否有明确的受众指向？
