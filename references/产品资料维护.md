# 产品资料维护

用于：添加新产品、更新产品信息、查询产品资料、管理产品库。

---

## 产品资料库位置

```
内容生产系统/01-素材库/产品资料库/
├── 产品索引.md              # 产品总览和快速链接
├── 产品标签体系.md           # 分类和标签定义
└── products/                # 产品资料文件
    ├── _模板.md             # 产品资料模板
    └── {产品名称}.md        # 单个产品资料
```

## 智能感知机制

### 产品信息自动检测

**检测信号：**
- 产品名称 + 价格（数字 + 元/块/￥）
- 产品名称 + 卖点（特点是/优势/功效）
- 产品名称 + 人群（适合/针对/用于）
- "新产品"/"上新"/"新增" + 产品描述

**处理策略：**

| 当前任务 | 检测策略 | 处理方式 |
|---------|---------|---------|
| 生成文稿 | 背景采集 | 不中断，自动提取信息并用于文案生成，完成后提示 |
| 记录选题 | 轻量确认 | 询问是否添加，1-2轮对话完成 |
| 检索素材 | 轻量确认 | 询问是否添加，1-2轮对话完成 |
| 自由对话 | 主动建议 | 检测到产品信息后主动建议维护 |

## 工作流

### 1. 添加新产品

**触发词：** 「新产品」、「添加产品」、「上新」、「录入产品」

**执行步骤：**
1. 询问产品基本信息：
   - 产品名称
   - 价格（原价/现价）
   - 核心卖点（3个）
   - 产品类别
2. 根据信息创建产品资料文件
3. 更新产品索引
4. 询问是否补充更多信息：
   - 目标人群
   - 使用场景
   - 用户证言
   - 文案素材

**信息采集模板：**
```markdown
## 新产品信息

**基础信息**
- 产品名称：
- 价格：¥{现价}（原价¥{原价}）
- 类别：□ 玄学类 □ 养生类 □ 家居类 □ 知识类 □ 其他

**核心卖点**（必须）
1. {卖点1}
2. {卖点2}
3. {卖点3}

**标签**（多选）
功效：□ 招财 □ 转运 □ 桃花 □ 健康 □ 睡眠 □ 事业 □ 学业 □ 平安
人群：□ 女性 □ 男性 □ 宝妈 □ 上班族 □ 创业者 □ 学生 □ 老人
价格：□ 入门 □ 中端 □ 高端 □ 奢华

**扩展信息**（可选）
- 目标人群：
- 解决痛点：
- 使用场景：
- 材质规格：
```

---

### 2. 智能背景采集（生成文案时）

**触发条件：** 用户在生成文案时提及新产品

**示例：**
```
用户：给"福运手链"写个带货文案，卖199，主要功效是招财转运

AI处理：
1. 检测到新产品信号：
   - 产品名称：福运手链
   - 价格：199
   - 卖点：招财转运

2. 检查产品库 → 不存在

3. 背景采集（不中断生成流程）：
   - 创建基础产品档案
   - 标记为"待完善"

4. 继续生成文案，使用采集到的信息

5. 生成完成后提示：
   "✅ 文案已生成。
   
   📝 检测到'福运手链'是新产品，已自动记录基础信息：
   - 名称：福运手链
   - 价格：199
   - 卖点：招财转运
   
   是否现在补充更多产品详情？（目标人群/使用场景/用户证言等）
   回复'是'继续完善，回复'否'稍后处理。"
```

---

### 3. 更新产品信息

**触发词：** 「更新产品」、「修改产品」、「补充产品信息」

**执行步骤：**
1. 询问要更新的产品名称
2. 读取现有产品资料
3. 显示当前信息，询问更新内容
4. 更新产品文件
5. 更新产品索引
6. 确认完成

---

### 4. 查询产品资料

**触发词：** 「查产品」、「找产品」、「产品资料」、「有哪些产品」

**执行步骤：**
1. 如指定产品名称 → 直接读取产品文件
2. 如未指定 → 显示产品索引
3. 支持按标签筛选：
   - 「找招财的产品」
   - 「有哪些入门产品」
   - 「适合宝妈的产品」

---

### 5. 产品使用记录

**自动记录：**
每次在文案中使用产品时，自动更新：
- `usage_count` +1
- 在使用记录表格中添加一行

**格式：**
```markdown
| 日期 | 文案标题 | 内容类型 | 效果数据 |
|------|----------|----------|----------|
| 2026-04-11 | 《标题》 | 投流视频 | 播放1.2w/点赞500 |
```

## 产品信息完整度评估

### 完整度等级

| 等级 | 完整度 | 状态 | 说明 |
|------|--------|------|------|
| 🟢 | 80%+ | 完整 | 可直接用于文案生成 |
| 🟡 | 50-80% | 可用 | 可以生成，建议补充 |
| 🔴 | <50% | 基础 | 需要补充关键信息 |

### 评分标准

| 字段 | 权重 | 分值 |
|------|------|------|
| 产品名称 | 必需 | 20 |
| 价格 | 高 | 15 |
| 核心卖点 | 高 | 15 |
| 目标人群 | 中 | 10 |
| 使用场景 | 中 | 10 |
| 差异化优势 | 中 | 10 |
| 用户证言 | 低 | 5 |
| 文案素材 | 低 | 5 |
| 使用记录 | 低 | 5 |
| 标签体系 | 中 | 5 |

## 文案生成时的产品调用

### 自动检索

当用户说「给XX产品写文案」时：
1. 提取产品名称
2. 检索产品资料库
3. 如存在 → 读取完整信息
4. 如不存在 → 启动智能采集

### 信息融合

**投流视频场景：**
```
开头钩子 → 结合产品解决的核心痛点
中段 → 融入产品卖点 + 用户收益 + 差异化优势
结尾 → 价格信息 + 促销话术 + CTA
```

**自然流带货场景：**
```
场景切入 → 使用产品的真实场景
体验描述 → 结合产品功能特点 + 用户证言
软性推荐 → 自然引出产品 + 使用场景
```

**口播视频场景：**
```
知识分享 → 结合产品相关的知识点
案例植入 → 产品作为案例或解决方案
价值输出 → 产品带来的改变和收益
```

### 占位符替换

| 占位符 | 替换内容 | 示例 |
|--------|---------|------|
| {产品名称} | 产品名称 | 福运手链 |
| {价格} | 当前价格 | 199 |
| {原价} | 原价 | 299 |
| {核心卖点} | 卖点列表 | 招财转运、天然水晶、手工编织 |
| {目标人群} | 人群描述 | 职场女性、创业者 |
| {使用场景} | 场景描述 | 日常佩戴、重要场合 |
| {用户证言} | 证言内容 | "戴了一个月，真的升职了" |
