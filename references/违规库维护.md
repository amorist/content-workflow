# 违规库维护

用于：查询违规规则、添加新违规案例、更新规避方法、审核内容合规性。

---

## 支持的违规库类型

1. **口播视频违规库** - 适用于利他内容、知识分享、种草类口播视频
2. **自然流带货违规库** - 适用于软性种草、场景化推荐、自然流带货视频
3. **投流视频违规库** - 适用于强转化、直接卖货、付费投放类视频

---

## 工作流

### 1. 查询违规规则

**触发词：** 「查违规」、「违规规则」、「这个能发吗」、「会不会违规」、「违规查询」

**执行步骤：**
1. 识别内容类型（口播视频/自然流带货/投流视频）
2. 如未指定，询问用户内容类型
3. 读取对应违规库文件
4. 根据用户查询关键词，提取相关违规规则
5. 提供规避方法和建议

**输出格式：**
```markdown
## 违规风险分析

**内容类型：** {口播视频/自然流带货/投流视频}

### 相关违规规则

#### {违规类别}
- **违规表现：** {具体表现}
- **规避方法：** {如何避免}

### 建议
{针对用户具体内容的合规建议}
```

---

### 2. 添加违规案例

**触发词：** 「添加违规」、「记录违规」、「新违规类型」、「补充违规规则」

**执行步骤：**
1. 询问内容类型和违规详情
2. 收集信息：
   - 违规类别（身份资质/产品宣传/内容真实/引流转化等）
   - 违规表现描述
   - 规避方法
   - 实际案例（如有）
3. 读取对应违规库文件
4. 在合适位置追加新违规规则
5. 更新记录时间
6. 确认添加完成

**信息收集模板：**
```markdown
## 新违规案例记录

**内容类型：** □ 口播视频 □ 自然流带货 □ 投流视频

**违规类别：** 
- □ 身份与资质类
- □ 产品宣传类  
- □ 内容真实性类
- □ 引流与转化类
- □ 其他：_____

**违规表现：**
{具体描述什么行为会触发违规}

**规避方法：**
{如何修改才能避免违规}

**实际案例（可选）：**
{具体的违规文案或场景}

**记录时间：** {YYYY-MM-DD}
```

---

### 3. 内容合规审核

**触发词：** 「审核内容」、「检查合规」、「文案合规检查」、「合规检查」

**执行步骤：**
1. 获取用户要审核的文案
2. 询问内容类型
3. 读取对应违规库的检查清单
4. 逐条检查文案是否存在违规风险
5. 标记风险点并提供修改建议
6. 输出合规检查报告

**审核报告格式：**
```markdown
## 内容合规检查报告

**内容类型：** {类型}
**审核时间：** {时间}

### 检查清单

- [ ] 身份资质类
  - [ ] 是否使用了虚假身份？ {结果}
  - [ ] ...

- [ ] 产品宣传类
  - [ ] 是否存在绝对化用语？ {结果}
  - [ ] ...

### 风险点

| 序号 | 风险类型 | 具体位置 | 风险等级 | 修改建议 |
|------|----------|----------|----------|----------|
| 1 | {类型} | {引用原文} | 高/中/低 | {建议} |

### 总体评估
{通过/需修改/高风险}
```

---

### 4. 更新规避方法

**触发词：** 「更新违规库」、「修改规避方法」、「优化违规规则」

**执行步骤：**
1. 询问要更新的违规库类型
2. 询问具体要更新的规则
3. 读取对应违规库文件
4. 定位到需要更新的规则条目
5. 更新规避方法或添加新的案例
6. 更新记录时间
7. 确认更新完成

## 违规类别体系

### 通用违规类别（适用于所有类型）

1. **身份与资质类**
   - 虚假身份营销
   - 无资质专业背书
   - 冒充官方身份

2. **产品宣传类**
   - 功效夸大
   - 虚假宣传
   - 绝对化用语

3. **内容真实性类**
   - 虚假知识/伪科学
   - 编造数据
   - 场景造假

4. **引流与转化类**
   - 违规引流
   - 隐性站外导流
   - 使用规避审核手段

### 各类型特殊违规

**口播视频特殊关注：**
- 健康/金融/法律等领域的专业资质
- 引用数据来源标注
- "仅供参考"免责声明

**自然流带货特殊关注：**
- 商业合作关系标注
- 真实使用场景
- 避免摆拍造假

**投流视频特殊关注：**
- 价格宣传真实性
- 促销时限真实性
- 产品资质范围
- 素材授权问题

## 质量标准

1. **准确性**：违规规则必须基于实际平台规则和真实案例
2. **实用性**：规避方法必须具体可操作
3. **时效性**：定期更新，标注记录时间
4. **完整性**：覆盖常见违规场景
