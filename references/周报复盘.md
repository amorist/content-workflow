# 周报复盘工作流

## 触发条件

- 「周报」
- 「这周怎么样」
- 「周报复盘」
- 「weekly retro」
- 每周五/周日主动建议

## 执行步骤

### Step 1: 运行前置检查

```bash
bash content-workflow/.content-workflow/preamble.sh
```

### Step 2: 收集本周数据

自动统计本周内容生产数据：

```bash
# 统计本周生产的文稿
WEEK_DRAFTS=$(find "内容生产系统/03-文稿库/草稿/" -name "*.md" -mtime -7 2>/dev/null | wc -l)
WEEK_FINAL=$(find "内容生产系统/03-文稿库/定稿/" -name "*.md" -mtime -7 2>/dev/null | wc -l)

# 统计本周时间线事件
WEEK_EVENTS=$(bash content-workflow/.content-workflow/timeline.sh today)

# 统计本周学习记录
if [ -f "content-workflow/.content-workflow/learnings.jsonl" ]; then
  WEEK_LEARNINGS=$(grep "$(date +%Y-%m-)" content-workflow/.content-workflow/learnings.jsonl | wc -l)
fi
```

### Step 3: 分析时间线

读取本周的时间线记录：

```bash
cat content-workflow/.content-workflow/timeline.jsonl | \
  grep "$(date +%Y-%m-)" | \
  tail -50
```

分析：
- 本周生产了多少内容？
- 平均每个内容的生产周期？
- 哪个环节耗时最长？

### Step 11: 保存周报

```bash
# 保存周报
WEEK_NUMBER=$(date +%U)
YEAR=$(date +%Y)
REPORT_FILE="内容生产系统/04-发布与数据/周报_${YEAR}_第${WEEK_NUMBER}周.md"

# 保存报告内容
cat > "$REPORT_FILE" << 'EOF'
{周报内容}
EOF

echo "✅ 周报已保存到: $REPORT_FILE"
```

## 复盘周期建议

| 周期 | 频率 | 重点 | 输出 |
|-----|------|------|------|
| 日报 | 每天 | 当日生产情况 | 简短总结 |
| 周报 | 每周 | 周趋势、模式分析 | 周报文档 |
| 月报 | 每月 | 月度总结、方向调整 | 月报文档 |
| 季报 | 每季 | 战略复盘、长期规划 | 战略文档 |

## 关键指标基准

### 健康度指标
| 指标 | 健康值 | 警告值 | 危险值 |
|-----|-------|-------|-------|
| 周发布量 | > 7 | 3-7 | < 3 |
| 爆款率 | > 10% | 5-10% | < 5% |
| 定稿率 | > 70% | 50-70% | < 50% |
| 平均播放 | > 5000 | 1000-5000 | < 1000 |

### 效率指标
| 指标 | 优秀 | 良好 | 需改进 |
|-----|------|------|-------|
| 平均生产周期 | < 2小时 | 2-4小时 | > 4小时 |
| 选题到发布 | < 1天 | 1-3天 | > 3天 |
| 素材复用率 | > 50% | 30-50% | < 30% |
