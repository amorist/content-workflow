# 数据复盘规范

## 数据记录模板

```markdown
## {标题}
- 发布日期：{YYYY-MM-DD}
- 发布平台：{视频号/抖音/小红书}
- 文稿链接：{path}

### 原始数据
- 播放量：{数字}
- 点赞数：{数字}
- 评论数：{数字}
- 收藏数：{数字}
- 分享数：{数字}

### 计算指标
- 点赞率：{点赞/播放 × 100%}%
- 互动率：{(点赞+评论+收藏+分享)/播放 × 100}%
- 爆款判定：{是/否}（播放>5000 或 点赞>500）

### 简短复盘
{2-3句话分析这次数据表现}

### 可复用点
- {学到什么？哪些做对了？}
```

## 爆款判定标准

| 平台 | 爆款门槛（参考） |
|------|----------------|
| 视频号 | 播放>1万，点赞>500 |
| 抖音 | 播放>10万，点赞>5000 |
| 小红书 | 浏览>5000，点赞>500 |

## 爆款拆解模板

当一条内容被判定为爆款时，自动生成：

```markdown
# 🔥 爆款拆解：{标题}

## 基本信息
- 播放量：{数字}
- 点赞率：{X}%
- 爆款原因：{推测的核心原因}

## 内容结构拆解

### 开头（前5秒）
{分析开头为什么吸引人}
- 类型：{痛点/好奇/数字/成果}
- 钩子：{具体文案}

### 主体结构
- 要点1：{内容} — {为什么有效}
- 要点2：{内容} — {为什么有效}
- 要点3：{内容} — {为什么有效}

### 结尾设计
{分析结尾的引导方式}

## 可复用规律

### 选题规律
{这次爆款的选题逻辑}

### 结构规律
{这次爆款的结构特点}

### 表达规律
{这次爆款的语言风格/表达方式}

## 方法论更新
{根据这条爆款，更新到方法论沉淀}
```

## 数据复盘流程

1. **收集数据**
   - 用户输入：「记录数据：标题/播放/点赞/评论/收藏」
   - 如果用户没有全部数据，先问清楚哪些有

2. **计算指标**
   - 点赞率 = 点赞/播放
   - 互动率 = (点赞+评论+收藏+分享)/播放

3. **判断是否爆款**
   - 爆款 → 触发爆款拆解流程
   - 非爆款 → 只记录 + 简短复盘

4. **更新方法论**
   - 分析：什么类型的选题/结构/表达方式数据好？
   - 更新：05-方法论沉淀/选题规律.md

## 数据反哺机制

### 内容数据记录.md 累积多了之后
定期（每周/每月）分析：
- 哪类选题播放最高？
- 哪种开头形式完播率最高？
- 什么时长最合适？

### 方法论迭代
根据数据，不断更新：
- 短视频标题方法论
- 短视频开头方法论
- 选题规律

**核心逻辑：不是凭感觉，是凭数据迭代。**
