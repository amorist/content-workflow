# 数据复盘工作流

## 触发条件

- 「数据复盘」
- 「分析效果」
- 「为什么没爆」
- 「这条数据怎么样」
- 用户发布内容后

## 执行步骤

### Step 1: 运行前置检查

```bash
bash content-workflow/.content-workflow/preamble.sh
```

### Step 2: 收集数据

询问用户以下数据（如果用户未提供）：

> 请提供以下数据：
> - **标题：** {内容标题}
> - **发布时间：** {发布日期时间}
> - **播放量：** {数字}
> - **点赞数：** {数字}
> - **评论数：** {数字}
> - **收藏数：** {数字}
> - **转发数：** {数字}
> - **完播率：** {百分比，如有}
> - **3秒完播率：** {百分比，如有}
> - **粉丝增长：** {数字，如有}

### Step 3: 计算核心指标

基于提供的数据，计算：

```markdown

| 指标 | 数值 | 行业均值 | 表现评级 |
|-----|------|---------|---------|

| 点赞数 | {X} | - | - |
| 评论数 | {X} | - | - |
| 收藏数 | {X} | - | - |
| 转发数 | {X} | - | - |

### 转化率计算
| 指标 | 计算方式 | 结果 | 行业均值 | 表现评级 |

| 收藏率 | 收藏/播放 | {X}% | 1-2% | {优秀/良好/一般/差} |
| 转发率 | 转发/播放 | {X}% | 0.3-0.5% | {优秀/良好/一般/差} |
| 互动率 | (点赞+评论+收藏)/播放 | {X}% | 5-8% | {优秀/良好/一般/差} |

| 流失点 | {分析} | - | - |
```

### Step 4: 表现诊断

根据数据表现，进行诊断：

#### 情况A: 播放量高，互动低
**诊断：** 开头吸引人，但内容价值不足
**可能原因：**
- 标题党，内容与预期不符
- 内容过于浅显，没有信息增量
- 缺乏情感共鸣或实用价值

#### 情况B: 播放量低，互动高
**诊断：** 内容质量好，但分发不足
**可能原因：**
- 开头不够抓人
- 标签/话题选择不当
- 发布时间不合适
- 封面不够吸引人

#### 情况C: 播放量低，互动低
**诊断：** 整体表现不佳
**可能原因：**
- 选题本身没有吸引力
- 开头平淡
- 内容结构混乱
- 与平台调性不符

#### 情况D: 播放量高，互动高
**诊断：** 爆款内容
**分析重点：**
- 哪些元素导致了成功？
- 能否复用这些元素？
- 如何进一步优化？

### Step 7: 提取学习

  "{主题}" \
  "{核心学习点}" \

  "{标签}"

### Step 9: 判断是否加入爆款拆解

如果内容表现优秀（满足以下任一条件）：
- 播放量 > 5000
- 点赞 > 500
- 互动率 > 10%
- 收藏率 > 5%

则：

```markdown
> 🎉 这条内容表现优秀！建议加入爆款拆解库，方便后续复用。
> 
> 是否现在生成爆款拆解？
> - [ ] 是，立即生成
> - [ ] 否，稍后处理
```

### Step 10: 记录时间线

```bash
bash content-workflow/.content-workflow/timeline.sh review-data "{标题}" "{播放量}"
```

## 特殊情况处理

**如果数据异常（如播放量突然暴涨/暴跌）：**
- 检查是否有外部因素（热点、推荐等）
- 分析时间分布，找出爆发点
- 对比同类内容的正常表现

**如果用户没有完播率数据：**
- 基于互动率推断内容质量
- 建议用户开通数据权限
- 使用评论内容分析用户反馈
